| 项目 | 内容 |
|------|------|
| **项目名称** | OpenClaw 21点 |

## 2. 游戏规则

### 2.1 基本规则

| 规则 | 说明 |
|------|------|
| 发牌 | 玩家和庄家各发2张牌 |
| 点数计算 | A=1或11，2-10=面值，J/Q/K=10 |
| 目标 | 手牌点数接近21且不超过21 |
| 爆牌 | 超过21点直接输掉 |

### 2.2 游戏流程

```
开始游戏 → 发放初始牌 → 玩家决策(要牌/停牌) → 庄家行动 → 比对结果 → 结算分数 → 记录战绩
```

### 2.3 操作说明

| 操作 | 说明 |
|------|------|
| 要牌 | 点击"Hit"按钮，额外获得一张牌 |
| 停牌 | 点击"Stand"按钮，结束玩家回合 |
| 新游戏 | 点击"New Game"开始下一局 |

### 2.4 计分规则

| 结果 | 分数 |
|------|------|
| 胜利 | +10分 |
| 失败 | -5分 |
| 平局 | 0分 |
| 爆牌 | -5分 |

## 5. 接口设计

### 5.1 API 列表

| 方法 | 路径 | 说明 | 请求体 | 响应 |
|------|------|------|--------|------|
| POST | /api/game/start | 开始新游戏 | {playerId} | {gameId, playerCards, dealerCards} |
| POST | /api/game/hit | 玩家要牌 | {gameId} | {playerCards, playerScore, status} |
| POST | /api/game/stand | 玩家停牌 | {gameId} | {dealerCards, result, score} |
| GET | /api/score/:playerId | 获取玩家分数 | - | {playerId, totalScore, wins, losses, draws} |

### 5.2 响应格式

```json
{
  "success": true,
  "data": {
    "gameId": "abc123",
    "playerCards": ["♠A", "♥10"],
    "playerScore": 21,
    "status": "playing"
  }
}
```

### 5.3 错误响应

```json
{
  "success": false,
  "error": {
    "code": "GAME_NOT_FOUND",
    "message": "游戏不存在或已结束"
  }
}
```
